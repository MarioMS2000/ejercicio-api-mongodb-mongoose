1- Crear repo nuevo

    1.1 Haz un repositorio nuevo en GitHub, por ejemplo:

        ejercicio-api-mongodb-mongoose

    Luego clónalo:

        git clone https://github.com/TU_USUARIO/ejercicio-api-mongodb-mongoose.git
        cd ejercicio-api-mongodb-mongoose

2- Inicializar Node
    npm init -y

3- Instalar dependencias

    La plantilla usa express, mongoose y dotenv, y el enunciado te pide Morgan. El package.json del repo plantilla también incluye nodemon.

    Instala:

        npm install express mongoose dotenv morgan
        npm install -D nodemon

    Y deja los scripts así:

        {
        "scripts": {
            "dev": "nodemon server.js",
            "start": "node server.js"
        }
        }

4- Variables de entorno

    Servidor → dotenv → variables → config app

    Crea el archivo .env:

        PORT=3000
        MY_MONGO_URI=mongodb://127.0.0.1:27017/ejercicio_api_rest

    Y .env.example:

        PORT=3000
        MY_MONGO_URI=

    La plantilla ya expone MY_MONGO_URI como variable de entorno esperada.

5- Conexión a MongoDB externalizada

    Servidor → config/db_mongo → MongoDB

    config/db_mongo.js

6- Modelado de datos

    Controller → Model → MongoDB → Model → Controller

    Aquí aplicamos la misma idea del demo de Game y Publisher: el producto guarda un ObjectId apuntando al proveedor, y luego se usa populate() para traer datos relacionados.

    models/Provider.js

7- Servicios auxiliares

    Controller → Service → Model → MongoDB → Model → Service → Controller

    Sirve para separar la lógica de acceso a la base de datos del resto de la aplicación.

        Encapsular consultas a la base de datos.
        Separar lógica (arquitectura limpia).
        Reutilizar código.
        Facilitar mantenimiento.
    
    provider.service.js

8- Controladores

    Cliente → Middleware → Ruta → Controller → Modelo → DB
                                   ↓
    Cliente ← Middleware ← Controller ← Modelo ← DB

    Aquí va la lógica de negocio real.
    Decide qué hacer cuando llega una petición (GET, POST, PUT, DELETE).
    
        controllers/providers.controller.js
        controllers/products.controller.js

9- Routes

    Cliente → Middleware → Ruta → Controller

    Aquí es donde conectas todo lo anterior (controllers) con las rutas HTTP
        Es decir: defines qué URL llama a qué función.
            routes/providers.routes.js
            routes/products.routes.js

10- Middlewares

    Middleware normal
        Cliente → Middleware → Ruta → Controller

    notFound (404)
        Cliente → Middleware → Ruta ❌ → notFound → Respuesta

    errorHandler (500)
    
    Cliente → Middleware → Ruta → Controller 💥 → next(error) → errorHandler → Respuesta

    Una función que se ejecuta entre la petición del cliente y la respuesta

    Flujo en tu API
        Cliente → Middleware → Ruta → Controller → Middleware → Respuesta
                middlewares/notFound.js
                middlewares/errorHandler.js

11- Servidor principal

    Server.js (flujo completo REAL)
        Cliente 
        ↓
        Middleware (express.json, morgan)
        ↓
        Ruta (/products, /providers)
        ↓
        Controller
        ↓
        Modelo (Mongoose)
        ↓
        MongoDB
        ↓
        Controller (respuesta)
        ↓
        Middleware (errorHandler si falla)
        ↓
        Respuesta al cliente

    Aquí es donde arranca tu servidor y se conecta todo. El archivo que pone en marcha tu API
        server.js

RESUMEN FINAL

Cliente → Middleware → Ruta → Controller → Modelo → DB
                                           ↓
Cliente ← Middleware ← Controller ← Modelo ← DB


12- PASO A PASO: Conectar con MongoDB Atlas
        1. Ir a MongoDB Atlas

        👉 https://www.mongodb.com/atlas

        Crear cuenta (si no tienes)
        Login

    2. Crear proyecto
        New Project
        Nombre: ejercicio-api (por ejemplo)
        Create

    3. Crear cluster
        Build a Database
        Elige:
        Free (M0) ✅
        Create cluster

        ⏳ Espera 1–2 minutos

    4. Crear usuario de base de datos (MUY IMPORTANTE)

        Te va a pedir:

        Username: por ejemplo admin
        Password: algo tipo 1234abcd

        👉 GUÁRDALO, lo necesitarás luego

    5. Permitir acceso a tu IP
        Network Access → Add IP
        Opción fácil:
        0.0.0.0/0

        👉 Permite acceso desde cualquier sitio (vale para el ejercicio)

    6. Obtener la URI de conexión
        Pulsa Connect
        Elige:
        👉 Drivers
        Copia algo como esto:
        mongodb+srv://admin:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority

    7. Modificar tu .env

        Cambia tu .env por esto:

        PORT=3000
        MY_MONGO_URI=mongodb+srv://admin:TU_PASSWORD@cluster0.xxxxx.mongodb.net/ejercicio_api_rest?retryWrites=true&w=majority

        🔥 IMPORTANTE
        Cambia TU_PASSWORD
        Añade el nombre de la DB:
        /ejercicio_api_rest

        👉 si no lo pones, Mongo no sabe qué base usar

    8. Reiniciar servidor

        Para todo:

        Ctrl + C

        Y vuelve a lanzar:

        npm run dev
        ✅ Resultado correcto

    Ahora deberías ver:

        ✅ Conectado a MongoDB
        🚀 Servidor corriendo en http://localhost:3000

13- Probar ya la API
